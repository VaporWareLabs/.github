# VaporWare Labs

An independent engineering lab for software, robotics, AI, embedded systems, home automation, infrastructure, and experimental projects.

🌐 https://www.vaporwarelabs.org

---

## About VaporWare Labs

VaporWare Labs is a growing collection of engineering projects, experiments, and shared knowledge.

Projects range from autonomous vehicles and smart-home systems to project-management software, local artificial intelligence, self-hosted infrastructure, and documented cooking experiments.

The guiding purpose is:

> **Experiment. Share Results. Enjoy the Process.**

VaporWare Labs is not limited to one technical field or product. It is a place to follow curiosity, build useful things, preserve what was learned, and make the results available to others.

## Active Projects

### 📊 vaporCAM

An open-source engineering planning and control-account management application built with Django.

vaporCAM provides a simplified operational layer over Jira for planning, forecasting, and earned-value visibility.

Current capabilities include:

- Jira Feature and Story synchronization
- Monthly ETC planning
- Labor actuals tracking
- Feature-level rollups
- Baseline and percent-complete visibility
- Search, filtering, notes, and audit history
- Excel export
- Linux deployment with Gunicorn and Nginx

Current focus:

- Production stabilization
- User-reported bug fixes
- Actuals-import refinement
- Excel round-trip import
- Performance and usability improvements

---

### 🧠 vaporAI

A private, self-hosted artificial-intelligence platform running on VaporWare Labs infrastructure.

Current stack:

- Ubuntu Server
- Ollama
- Open WebUI
- Local language models
- Retrieval-Augmented Generation (RAG)

vaporAI can retrieve information from the `vaporware-knowledge` documentation repository and answer questions about VaporWare Labs projects using locally stored sources.

Current focus:

- Improving RAG quality
- Expanding project documentation
- Evaluating local models
- Automating knowledge synchronization
- Developing a private engineering-memory assistant

---

### 📚 vaporware-knowledge

A version-controlled knowledge repository designed for both humans and AI systems.

It contains:

- Project overviews
- Design documents
- Roadmaps
- Feature descriptions
- Frequently asked questions
- Infrastructure notes
- Experiments and lessons learned
- Recipes and project history

The repository serves as the curated institutional memory behind vaporAI.

---

### 🍳 vaporCHEF

A Git-based collection of cooking experiments and evolving recipes documented in Markdown.

The project applies an engineering mindset to food:

- Record the initial recipe
- Test it
- Evaluate the result
- Revise the ingredients or method
- Preserve the version history

Initial recipes include VaporWare Labs Black Pepper Pineapple Chicken Fried Rice.

## Design Philosophy

Projects are developed using an iterative engineering approach:

1. Start with a useful experiment
2. Build the smallest workable version
3. Test it in the real world
4. Study the results
5. Improve the design
6. Document what was learned
7. Share the results

Documentation is treated as a first-class engineering artifact. Whenever practical, projects preserve not only what was built, but also why important decisions were made.

## Organization Philosophy

VaporWare Labs is intended to operate as a loose community of makers, engineers, experimenters, and collaborators.

It is not defined by ownership or hierarchy. The goal is to create an open ecosystem where useful ideas can be developed, documented, improved, and shared.

A VaporWare Labs mark represents recognition of the craftsmanship behind something that was thoughtfully made.
